[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture on our workflow diagram. 
First, when a user sends a query, it enters the Security Checkpoint node. This node scrubs PII and checks for prompt injections. 
If it is clean, the query routes to the Orchestrator node, which activates the orchestrator_agent.
If cryptographic operations are requested, the orchestrator_agent delegates to the crypto_agent using an AgentTool. The crypto_agent performs encryption and decryption locally in memory.
For database reads and writes, the orchestrator_agent calls the db_agent, which uses an MCP server toolset to save partitions and write cryptographically linked log blocks to the audit ledger.
And finally, the blue node represents our Human-in-the-loop gate, where the orchestrator pauses and asks the user for their master password to authorize vault decryption.
